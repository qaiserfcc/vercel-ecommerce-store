NEXT STEPS - DEPLOY TO VERCEL:
=====================================

IMPORTANT: You must be authenticated with Vercel to proceed.

Option A - Via Dashboard (RECOMMENDED):
1. Go to https://vercel.com
2. Click "Add New Project"
3. Import: qaiserfcc/vercel-ecommerce-store
4. Configure:
   - Framework: Other
   - Build Command: npm install && cd client && npm install --legacy-peer-deps && npm run build
   - Output Directory: client/build
5. Click Deploy
6. Add Environment Variables in Vercel Dashboard:
   - DATABASE_URL (PostgreSQL connection string)
   - JWT_SECRET (random hex string)
   - NODE_ENV=production

Option B - Via CLI:
1. vercel login
2. cd to project root
3. vercel --prod
4. Configure environment variables when prompted

=====================================
REQUIRED BEFORE DEPLOYMENT:
=====================================

1. DATABASE SETUP
   Choose a PostgreSQL provider:
   - Neon (neon.tech) - Recommended
   - Supabase (supabase.com)
   - Railway (railway.app)
   - Render (render.com)

2. DATABASE INITIALIZATION
   After getting connection string:
   psql "postgresql://..." -f database/schema.sql
   psql "postgresql://..." -f database/seed.sql (optional)

3. JWT SECRET
   Generate: openssl rand -hex 32
   Or: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

=====================================
TROUBLESHOOTING:
=====================================

If port 3000 is in use:
  lsof -i :3000

If build fails with TypeScript:
  npm install --legacy-peer-deps --force

If database connection fails:
  Verify: postgresql://username:password@host:port/database
  Check firewall/network rules
